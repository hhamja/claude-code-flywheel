# 백로그 작성 규칙

1. **Verify 먼저** — 티켓마다 "통과를 증명하는 셸 명령"을 먼저 쓰고, 그것을 통과시키는
   작업으로 역산하라. Verify 를 못 쓰겠으면 티켓이 아직 모호하다는 뜻이다.
2. **1티켓 = 1사이클** — 워커 1회 호출로 끝나는 분량. 크면 쪼개라.
3. **의존 순서 = 파일명 번호** — 001 이 002 보다 먼저 실행된다. 나중에 티켓을 끼워 넣을
   수 있게 번호 간격(010, 020, …)을 띄우는 것도 좋다.
4. **셸로 검증 불가능하면 티켓 자격이 없다** — 더 쪼개거나, 사람 판단이 필요한 일이면
   티켓이 아니라 GOAL.md 검토 요청으로 사람에게 보고하라.
5. 형식은 loop-protocol 스킬의 `references/ticket-format.md` 를 따르라.

좋은/나쁜 예시가 필요하면 `references/examples.md` 를 읽어라.
